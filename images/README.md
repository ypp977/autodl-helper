## Release rule

截图在公开发布前必须人工检查，确认不包含以下内容：

- 真实手机号
- token / cookie / authorization header
- 实例 ID、订单信息或账号页敏感信息
- 本地路径、邮件地址、个人隐私信息

如果截图尚未完成脱敏，不建议直接嵌入 README。

## Naming rule

后续如果继续补截图，建议使用更有语义的文件名，例如：

- `interactive-home.png`
- `interactive-diagnostics.png`
- `keeper-plan.png`
- `scheduled-progress.png`

如果暂时还没完成统一重命名，也至少保证：

1. 文件名稳定
2. README 引用路径稳定
3. 每张截图都能说明用途
